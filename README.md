# miniGPT
> 手搓极简版miniGPT，实现风格化文本续写，深入理解GPT底层原理  
> Build a minimal miniGPT from scratch, implement stylized text continuation, and deeply understand the underlying principles of GPT

## 1. 项目介绍 / Introduce

以ChatGPT为代表的GPT类生成式人工智能近年来热度极高，开启了全民AI时代，现今主流的语言大模型也普遍沿用其核心结构。

本项目基于PyTorch从零搭建一个极简版miniGPT（参考前置项目：TransDemo），不直接调用封装好的高级模型接口，专注于理解解码器结构、因果注意力、自回归生成等核心机制。

本项目以约 40 万字小说作为训练样本，实现该小说风格下的文本续写任务，覆盖模型搭建、数据处理、训练与推理全流程，做到理论与实战结合，深入掌握 GPT 类模型的底层原理。

## 3. 训练 / Training

#### 训练相关信息：
- 使用约41万字的中文小说作为训练数据
- Batch Size设置为32，BLOCK为128，共训练6万个step：
- 采用AdamW优化器：lr=5e-5, betas=(0.9, 0.98)
- 采用分组weight_decay：0.03 & 0.0
- 采用组合调度器：LinearLR（做预热）+ CosineAnnealingLR（余弦退火）+ ConstantLR （最后观察）
- 采用混合精度：autocast（FP16）+ GradScaler
- 采用自定义软标签交叉熵损失为loss，smoothing=0.1

## 5. 实现介绍 / Implementation Introduction
>  前置项目为：TransDemo，本项目介绍以差异为主，具体实现请结合参考前置项目以及代码

#### 1.  GPTLayer：
- 基于Transformer Decoder Layer的演进
- 由Post-LN改为Pre-LN
- 仅做一次多头注意力（自注意力）
- 采用自回归序列，预测Next Token
- 因果掩码（可复用原Transformer）
- FFN的激活函数由ReLU升级为GELU

#### 2.  MiniGPT：
- 主体通过nn.ModuleList组装多层GPTLayer构成
- 位置编码，由固定，改为可学习的Embedding
- 主体输出后，由于Pre-LN，需再接一次Norm
- 最后通过FC（nn.Linear）输出

#### 3.  数据处理：
- 使用正则简单清理数据（数据本身质量良好）
- 文本换行处，通过插入<SEP>衔接，形成长文本（id list）
- 提前列举并存储在长文本（id list）中所有的有效起始索引位置（标点符号或特殊标记后）
- Batch取数据时，从有效索引位置随机取起始id，然后截BLOCK_SIZE+1长度的id序列

#### 4.  词表：
- 对初步处理的训练数据，直接按字切分，然后去重，得到不重复字（含标点） 3278个
- 统计各个字在数据集中的出现频率，如：5次以上的2426、5次以上的2168、10次以上的1748、20次以上的1340
- 简单人工排查处于边界值的字，如出现频率为9的字，是否为典型或重要常用字
- 最终选定低频字过滤阈值：```UNK_THRESHOLD = 8```，过滤后词表压缩至```VOCAB_SIZE=1973```，低频字用标志位<UNK>代替

#### 5.  推理：
- 封装3种推理策略：贪心、 Top K、 Top P
- 贪心：每次预测的Next Token均选择概率最高的Token
- Top K：根据预测的Next Token的概率先选取Top K，然后通过torch.multinomial采样
- Top P：根据预测的Next Token的概率，按高低排序，取累积概率到p的所有Token，进行torch.multinomial采样
- 温度： 对模型输出的logits先除以temperature，后进行Softmax，通过影响分布概率，控制输出多样性


## 6. 过程总结 / Conclusion

- 使用余弦退火可以很好的调整学习率，使模型快速收敛
- 从保存的最佳loss，以及loss曲线看，实际在2w步左右已经可以停止训练，后继优化不大
- 正确的使用温度值和搜索策略，有助于推理结果质量
